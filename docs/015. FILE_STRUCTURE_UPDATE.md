## ğŸ“ File Naming Convention

### File Names
- Format: `day-XX.ts` (where XX is 01-50)
- Examples: `day-01.ts`, `day-15.ts`, `day-50.ts`
- Always use **two digits** (01, not 1)

### Export Names
- Format: `dayXX` (where XX is 01-50)
- Examples: `day01`, `day15`, `day50`
- Always use **two digits** (day01, not day1)

## ğŸ“ Need Help?

### Common Questions

**Q: Do I need to edit `jobSearchData.ts`?**
A: No! Only edit the individual day files in `src/data/days/`

**Q: What if I want to add a new day?**
A: Create a new file (e.g., `day-51.ts`), then add it to the imports and array in `jobSearchData.ts`

**Q: Can I rename the day files?**
A: Not recommended. The naming convention (day-01.ts, day-02.ts, etc.) keeps things organized

**Q: How do I know which days need content?**
A: Days 1-30 are complete. Days 31-50 have placeholder content that needs to be replaced

**Q: What if I break something?**
A: Run `npm run lint` to check for errors. TypeScript will tell you exactly what's wrong
