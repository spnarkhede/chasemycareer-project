# Login Troubleshooting Guide

## üîß Quick Fixes

### Issue: Can't Type in Email or Password Fields

**Symptoms:**
- Email field is grayed out
- Password field is grayed out
- Can't click or type in the fields

**Causes:**
- Form is stuck in loading state
- Previous login attempt didn't complete

**Solutions:**

1. **Refresh the page**
   - Press `F5` or `Ctrl+R` (Windows/Linux)
   - Press `Cmd+R` (Mac)
   - This will reset the form state

2. **Clear browser cache**
   - Press `Ctrl+Shift+Delete` (Windows/Linux)
   - Press `Cmd+Shift+Delete` (Mac)
   - Select "Cached images and files"
   - Click "Clear data"

3. **Try a different browser**
   - Chrome, Firefox, Safari, or Edge
   - Sometimes browser extensions can interfere

4. **Check browser console**
   - Press `F12` to open developer tools
   - Click "Console" tab
   - Look for error messages
   - Share errors with support if needed

---

### Issue: "Signing in..." Button Stuck

**Symptoms:**
- Button shows "Signing in..." indefinitely
- Spinner keeps spinning
- Nothing happens after clicking

**Causes:**
- Network timeout
- Server temporarily unavailable
- IP address fetch hanging

**Solutions:**

1. **Wait 10 seconds**
   - The system now has a 3-second timeout
   - Should auto-recover within 10 seconds
   - If not, proceed to next step

2. **Refresh the page**
   - Press `F5` or `Ctrl+R`
   - Try logging in again

3. **Check your internet connection**
   - Make sure you're connected to the internet
   - Try opening another website
   - Restart your router if needed

4. **Check browser console for errors**
   - Press `F12`
   - Look for red error messages
   - Common errors:
     - "Failed to fetch" ‚Üí Network issue
     - "Timeout" ‚Üí Server slow/unavailable
     - "Rate limited" ‚Üí Too many attempts

### Issue: Login Succeeds But Stays on Login Page

**Symptoms:**
- Success message appears
- But page doesn't redirect
- Still on login page

**Causes:**
- JavaScript error during redirect
- Browser blocking navigation
- Session not properly set

**Solutions:**

1. **Manually navigate to dashboard**
   - Change URL to: `https://yourdomain.com/dashboard`
   - Or click the logo to go home

2. **Check if you're actually logged in**
   - Look for your profile picture in header
   - Try accessing a protected page
   - Check if "Sign In" button is still visible

3. **Clear cookies and try again**
   - Press `Ctrl+Shift+Delete`
   - Select "Cookies and other site data"
   - Click "Clear data"
   - Try logging in again

4. **Disable browser extensions**
   - Some extensions block redirects
   - Try in incognito/private mode
   - Disable ad blockers temporarily

### Issue: "Too Many Failed Attempts" Error

**Symptoms:**
- Error message: "Too many failed attempts. Please try again in 15 minutes."
- Can't log in even with correct password

**Causes:**
- Multiple failed login attempts
- Rate limiting protection activated

**Solutions:**

1. **Wait 15 minutes**
   - This is a security feature
   - Protects against brute force attacks
   - Timer resets after 15 minutes

2. **Check your password**
   - Make sure Caps Lock is off
   - Check for extra spaces
   - Try resetting password if unsure

3. **Try from a different network**
   - Rate limit is per IP address
   - Use mobile data instead of WiFi
   - Or try from a different location

4. **Contact support**
   - If you're locked out unfairly
   - Support can manually reset the limit

---

### Issue: "Invalid Email or Password" Error

**Symptoms:**
- Error message appears immediately
- Sure the credentials are correct

**Causes:**
- Typo in email or password
- Account doesn't exist
- Password was changed

**Solutions:**

1. **Double-check your email**
   - Make sure it's the correct email
   - Check for typos
   - No extra spaces

2. **Check Caps Lock**
   - Passwords are case-sensitive
   - Make sure Caps Lock is off

3. **Try password reset**
   - Click "Forgot Password" (if available)
   - Or contact support

4. **Check if account exists**
   - Try signing up with the same email
   - If it says "Email already exists", account is there
   - If not, you may need to create an account

---

### Issue: Google Sign-In Not Working

**Symptoms:**
- "Continue with Google" button does nothing
- Popup window closes immediately
- Error message appears

**Causes:**
- Popup blocked by browser
- Google OAuth not configured
- Third-party cookies disabled

**Solutions:**

1. **Allow popups**
   - Look for popup blocker icon in address bar
   - Click and select "Always allow popups"
   - Try again

2. **Enable third-party cookies**
   - Go to browser settings
   - Search for "cookies"
   - Enable "Allow third-party cookies"
   - Or add exception for this site

3. **Try in incognito mode**
   - Press `Ctrl+Shift+N` (Chrome)
   - Press `Ctrl+Shift+P` (Firefox)
   - Try Google sign-in again

4. **Use email/password instead**
   - If Google sign-in keeps failing
   - Use regular email/password login
   - Can link Google account later

## üîç Debugging Steps

### Step 1: Check Browser Console

1. Press `F12` to open developer tools
2. Click "Console" tab
3. Look for these messages:

**Good signs (login working):**
```
Form submitted with email: user@example.com
Calling authApi.signIn...
Starting sign-in process for: user@example.com
Calling Supabase auth...
Sign-in successful, recording session...
Sign-in complete, returning user
Navigating to dashboard...
```

**Bad signs (login failing):**
```
Failed to get client IP: [error]
Rate limit check error: [error]
Sign-in error: [error]
Failed to record session: [error]
```

### Step 2: Check Network Tab

1. Press `F12` to open developer tools
2. Click "Network" tab
3. Try logging in
4. Look for failed requests (red)

**Common issues:**
- `api.ipify.org` - Timeout or failed (not critical)
- `supabase.co` - Failed (critical - check internet)
- `check_rate_limit` - Failed (not critical)
- `signInWithPassword` - Failed (critical - check credentials)

### Step 3: Check Application Tab

1. Press `F12` to open developer tools
2. Click "Application" tab
3. Look at "Local Storage"
4. Check for authentication tokens

**What to look for:**
- `supabase.auth.token` - Should exist after login
- If missing, login didn't complete
- If present, you're logged in

- Clear cache: `Ctrl+Shift+Delete`
- Disable extensions: `edge://extensions`
- Reset settings: `edge://settings/reset`

## üÜò When to Contact Support

Contact support if:

1. **Login fails consistently**
   - Tried all troubleshooting steps
   - Multiple browsers don't work
   - Different networks don't work

2. **Account locked**
   - Rate limited for too long
   - Can't reset password
   - Account suspended

3. **Technical errors**
   - Console shows critical errors
   - Database errors
   - Server errors (500, 502, 503)

4. **Security concerns**
   - Unauthorized access
   - Suspicious activity
   - Account compromised

---

## üìß What to Include in Support Request

When contacting support, include:

1. **Your email address** (for account lookup)
2. **Error message** (exact text)
3. **Browser and version** (e.g., Chrome 120)
4. **Operating system** (e.g., Windows 11)
5. **Console errors** (screenshot or copy/paste)
6. **Steps to reproduce** (what you did)
7. **When it started** (date/time)
8. **What you've tried** (troubleshooting steps)

## ‚úÖ Prevention Tips

### Keep Login Working Smoothly:

1. **Use a modern browser**
   - Chrome, Firefox, Safari, or Edge
   - Keep browser updated

2. **Stable internet connection**
   - Wired connection is best
   - Strong WiFi signal
   - Avoid public WiFi for sensitive operations

3. **Enable cookies**
   - Required for authentication
   - Don't block third-party cookies

4. **Don't use VPN for login**
   - Can cause rate limiting
   - May trigger security checks
   - Use only if necessary

5. **Remember your password**
   - Use a password manager
   - Write it down securely
   - Don't share with others

6. **Keep account secure**
   - Use strong password
   - Enable 2FA if available
   - Don't reuse passwords
