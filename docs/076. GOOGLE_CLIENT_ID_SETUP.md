# Google Client ID Setup Guide

## üéØ Where to Add Your Google Client ID

Your application now uses **@react-oauth/google** for authentication. Follow these steps to configure it:

---

## üìç Step 1: Get Your Google Client ID

### Create OAuth 2.0 Credentials

1. Go to [Google Cloud Console](https://console.cloud.google.com)
2. Create a new project or select an existing one
3. Navigate to **APIs & Services** ‚Üí **Credentials**
4. Click **Create Credentials** ‚Üí **OAuth client ID**
5. If prompted, configure the OAuth consent screen first
6. Select **Web application** as the application type
7. Give it a name (e.g., "Chase My Career Web Client")

### Configure Authorized Origins and Redirect URIs

**Authorized JavaScript origins:**
```
http://localhost:5173
http://localhost
https://your-production-domain.com
```

**Authorized redirect URIs:**
```
http://localhost:5173
http://localhost:5173/login
https://your-production-domain.com
https://your-production-domain.com/login
```

8. Click **Create**
9. **Copy your Client ID** (it looks like: `123456789-abc123.apps.googleusercontent.com`)

---

## üìç Step 2: Add Client ID to Your Application

### Option 1: Environment Variable (Recommended)

1. Open the `.env` file in the root of your project
2. Find this line:
   ```env
   VITE_GOOGLE_CLIENT_ID=YOUR_GOOGLE_CLIENT_ID_HERE.apps.googleusercontent.com
   ```
3. Replace `YOUR_GOOGLE_CLIENT_ID_HERE.apps.googleusercontent.com` with your actual Client ID
4. Save the file
5. Restart your development server

**Example:**
```env
VITE_GOOGLE_CLIENT_ID=123456789-abc123def456.apps.googleusercontent.com
```

### Option 2: Direct in Code (Not Recommended for Production)

If you want to hardcode it (only for testing), edit `src/main.tsx`:

```typescript
// Replace this line:
const googleClientId = import.meta.env.VITE_GOOGLE_CLIENT_ID || '';

// With your actual Client ID:
const googleClientId = '123456789-abc123def456.apps.googleusercontent.com';
```

‚ö†Ô∏è **Warning:** Don't commit hardcoded credentials to version control!

## üé® How It Works

### Authentication Flow

```
1. User clicks "Sign In with Google" button
   ‚Üì
2. Google OAuth popup appears
   ‚Üì
3. User selects Google account and grants permissions
   ‚Üì
4. Google returns a credential (JWT token)
   ‚Üì
5. Application decodes the token to get user info
   ‚Üì
6. User info stored in localStorage
   ‚Üì
7. User redirected to calendar
   ‚Üì
8. User is now logged in!
```

### User Data Storage

- **Where:** Browser's localStorage
- **What:** User's name, email, profile picture, and Google ID
- **Duration:** Until user signs out or clears browser data
- **Security:** Client-side only, no server storage

### Sign Out Flow

```
1. User clicks "Sign Out" button
   ‚Üì
2. Google session cleared (googleLogout())
   ‚Üì
3. User data removed from localStorage
   ‚Üì
4. User redirected to landing page
   ‚Üì
5. User is now signed out!
```

## üîí Security Notes

### What's Stored

The application stores the following in localStorage:
- Email address
- Full name
- Profile picture URL
- Google user ID (sub)

### What's NOT Stored

- No passwords
- No access tokens
- No refresh tokens
- No sensitive data

### Privacy

- User data is only stored locally in the browser
- No data is sent to any server
- Data is cleared when user signs out
- Data persists across page refreshes for convenience

## üêõ Troubleshooting

### Error: "Invalid Client ID"

**Problem:** The Client ID is incorrect or not set

**Solution:**
1. Check that you copied the entire Client ID from Google Console
2. Make sure there are no extra spaces or characters
3. Verify the `.env` file is in the root directory
4. Restart your development server after changing `.env`

### Error: "redirect_uri_mismatch"

**Problem:** The redirect URI doesn't match what's configured in Google Console

**Solution:**
1. Go to Google Cloud Console ‚Üí Credentials
2. Edit your OAuth client
3. Add `http://localhost:5173` to Authorized JavaScript origins
4. Add `http://localhost:5173` to Authorized redirect URIs
5. Save and try again

### Error: "Access blocked: This app's request is invalid"

**Problem:** OAuth consent screen not properly configured

**Solution:**
1. Go to Google Cloud Console ‚Üí OAuth consent screen
2. Fill in all required fields:
   - App name
   - User support email
   - Developer contact email
3. Add test users (your email) if app is in testing mode
4. Save and try again

### Google Sign-In Button Not Showing

**Problem:** Client ID not loaded or invalid

**Solution:**
1. Open browser console (F12)
2. Check for error messages
3. Verify `VITE_GOOGLE_CLIENT_ID` is set in `.env`
4. Make sure you restarted the dev server
5. Clear browser cache and try again

### Session Not Persisting

**Problem:** localStorage not working

**Solution:**
1. Check browser allows localStorage
2. Try in incognito mode to rule out extensions
3. Check browser console for errors
4. Clear browser data and try again

## üöÄ Production Deployment

### Before Deploying

1. **Add Production Domain to Google Console:**
   - Go to Google Cloud Console ‚Üí Credentials
   - Edit your OAuth client
   - Add your production domain to Authorized JavaScript origins:
     ```
     https://your-production-domain.com
     ```
   - Add your production domain to Authorized redirect URIs:
     ```
     https://your-production-domain.com
     https://your-production-domain.com/login
     ```
   - Save changes

2. **Set Environment Variable in Production:**
   - Add `VITE_GOOGLE_CLIENT_ID` to your hosting platform's environment variables
   - Use the same Client ID from Google Console
   - Rebuild your application

3. **Publish OAuth Consent Screen (if needed):**
   - Go to Google Cloud Console ‚Üí OAuth consent screen
   - Click "Publish App" if you want any Google user to sign in
   - Otherwise, add specific test users

- [ ] Rebuilt application with production settings
