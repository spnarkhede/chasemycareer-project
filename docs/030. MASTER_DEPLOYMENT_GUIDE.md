## Phase 3: Build & Deploy

### Option A: Deploy to Vercel (Recommended)

#### Step 3A.1: Push to GitHub

```bash
cd /workspace/chasemycareer

# Initialize git (if not already done)

git commit -m "Initial commit - ready for production"
git branch -M main

# Create GitHub repository and push
git remote add origin https://github.com/YOUR_USERNAME/chase-my-career.git
git push -u origin main
```

#### Step 3A.2: Import to Vercel

1. Go to https://vercel.com/dashboard
2. Click "Add New" â†’ "Project"
3. Import your GitHub repository
4. Configure:
   - Framework: Vite
   - Root Directory: `frontend`
   - Build Command: `npm run build`
   - Output Directory: `dist`

#### Step 3A.3: Set Environment Variables

In Vercel Dashboard â†’ Settings â†’ Environment Variables:

VITE_SUPABASE_ANON_KEY=your-anon-key

Add for: Production, Preview, Development

#### Step 3A.4: Deploy

1. Click "Deploy"
2. Wait ~2-3 minutes
3. Your app is live at: `https://your-project.vercel.app`

2. Search for: `chasemycareer.com`
3. Purchase domain (~$15/year)

### Step 4.2: Configure DNS (Vercel)

In your domain registrar's DNS settings:

**A Record:**
```

Name: @
Value: 76.76.21.21

```

**CNAME Record:**
```

Name: www
Value: cname.vercel-dns.com
TTL: 3600
```

## Troubleshooting

### Build Fails

```bash
# Clear cache and rebuild
rm -rf node_modules package-lock.json
npm install
npm run build
```

### Environment Variables Not Working

1. Verify variables are set in hosting dashboard
2. Ensure names start with `VITE_`
3. Redeploy after adding variables

### Domain Not Resolving

1. Check DNS records are correct
2. Wait 24-48 hours for propagation
3. Clear DNS cache: `ipconfig /flushdns`

### SSL Certificate Error

1. Wait 24 hours for provisioning
2. Verify DNS records
3. Contact hosting support

### Database Connection Error

1. Verify Supabase URL and key
2. Check Supabase project is not paused
3. Verify RLS policies are correct
