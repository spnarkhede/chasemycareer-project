## üìù DPA Requirements Checklist

When reviewing any DPA, ensure it includes:

### Essential Clauses

- [ ] **Subject Matter:** Clear description of data processing activities
- [ ] **Duration:** Term of the agreement
- [ ] **Nature and Purpose:** Why data is being processed
- [ ] **Type of Personal Data:** What data is being processed
- [ ] **Categories of Data Subjects:** Who the data is about (users)
- [ ] **Processor Obligations:** What the processor must do
- [ ] **Controller Rights:** Your rights as the data controller
- [ ] **Sub-processors:** List of any sub-processors used
- [ ] **Data Security:** Security measures in place
- [ ] **Data Breach Notification:** Process for breach notification
- [ ] **Data Subject Rights:** How to handle user rights requests
- [ ] **Data Deletion:** Process for data deletion
- [ ] **Audit Rights:** Your right to audit the processor
- [ ] **Data Transfer:** Mechanisms for international data transfers
- [ ] **Standard Contractual Clauses:** If data leaves the EU

## üîç Verification Steps

### Before Production Launch

1. **Identify All Processors:**
   - [ ] List all services that process user data
   - [ ] Categorize by data processing activities
   - [ ] Determine which require DPAs

2. **Review DPAs:**
   - [ ] Obtain DPA from each processor
   - [ ] Review for GDPR compliance
   - [ ] Check for SCCs if needed
   - [ ] Verify security measures

3. **Sign DPAs:**
   - [ ] Execute DPA with each processor
   - [ ] Keep signed copies on file
   - [ ] Document effective dates
   - [ ] Set calendar reminders for renewals

4. **Update Documentation:**
   - [ ] Update Privacy Policy with processor information
   - [ ] Document data flows
   - [ ] Create processor register
   - [ ] Train team on DPA requirements

5. **Ongoing Compliance:**
   - [ ] Review DPAs annually
   - [ ] Monitor for processor changes
   - [ ] Update Privacy Policy as needed
   - [ ] Maintain audit trail

## üö® Data Breach Protocol

In case of a data breach by a processor:

### Processor Obligations
1. **Notify you without undue delay** (typically within 24-48 hours)
2. **Provide details** of the breach
3. **Assist** with breach investigation
4. **Cooperate** with authorities

### Your Obligations
1. **Assess the breach** within 72 hours
2. **Notify supervisory authority** if required
3. **Notify affected users** if high risk
4. **Document** the breach and response
