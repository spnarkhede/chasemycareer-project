## üîê User Rights Implementation

### GDPR Rights Covered

#### 1. Right to Access
**Implementation:** Privacy Settings page displays all stored user data
- Name
- Email
- Google ID
- Profile picture URL

#### 2. Right to Data Portability
**Implementation:** Export Data feature
- Downloads all user data as JSON file
- Includes export date and timestamp
- Machine-readable format
- One-click download

**Code Example:**
```typescript
const handleExportData = () => {
  const userData = {
    exportDate: new Date().toISOString(),
    user: {
      name: user?.name || '',
      email: user?.email || '',
      picture: user?.picture || '',
      googleId: user?.sub || '',
    },
    localStorage: {
      authUser: localStorage.getItem('authUser'),
    },
  };
  // Download as JSON file
};
```

#### 3. Right to Erasure (Right to be Forgotten)
**Implementation:** Delete Data feature
- Removes all data from localStorage
- Signs user out
- Confirmation dialog to prevent accidental deletion
- Immediate effect

**Code Example:**
```typescript
const handleDeleteData = async () => {
  localStorage.removeItem('authUser');
  await signOut();
  navigate('/');
};
```

#### 4. Right to Withdraw Consent
**Implementation:** Sign out functionality
- Available in header menu
- Clears session
- Redirects to landing page

#### 5. Right to Object
**Implementation:** Cookie preferences
- Users can reject analytics cookies
- Users can customize cookie preferences
- Preferences respected across sessions

## üç™ Cookie Consent

### Cookie Categories

#### 1. Necessary Cookies
**Status:** Always enabled (cannot be disabled)
**Purpose:** Essential for website functionality
**Examples:**
- Session management
- Authentication tokens
- Security features

#### 2. Functional Cookies
**Status:** Optional (user can enable/disable)
**Purpose:** Enhanced functionality and personalization
**Examples:**
- Language preferences
- Theme settings
- UI customization

#### 3. Analytics Cookies
**Status:** Optional (user can enable/disable)
**Purpose:** Understanding user behavior
**Examples:**
- Page views
- User interactions
- Performance metrics

### Consent Flow

1. **First Visit:**
   - Cookie banner appears after 1 second
   - User sees three options: Accept All, Reject All, Customize

2. **Accept All:**
   - All cookie categories enabled
   - Consent saved to localStorage
   - Banner dismissed

3. **Reject All:**
   - Only necessary cookies enabled
   - Consent saved to localStorage
   - Banner dismissed

4. **Customize:**
   - Dialog opens with detailed cookie information
   - User can toggle functional and analytics cookies
   - Necessary cookies always enabled
   - Save preferences button

5. **Subsequent Visits:**
   - Banner does not appear
   - Preferences loaded from localStorage
   - User can change preferences anytime

### Storage

**Key:** `cookie-consent`  
**Value:** `'accepted'` or `'rejected'`

**Key:** `cookie-preferences`  
**Value:** JSON object with preferences
```json
{
  "necessary": true,
  "functional": true,
  "analytics": false
}
```
